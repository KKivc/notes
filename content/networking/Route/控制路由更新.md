路由策略
操作对象：路由条目
直接作用与路由表
分类
重分布redistribute：不同的路由协议间的通告
命令 //在不同协议临界的路由器操作 to where router where<静态，直连特殊>
---从ospf to rip---
- router rip
- redistribute ospf 1<area> metric 3 <rip metric默认无穷大 要设置>

---进ospf---
- router ospf 1
- redistribute eigrp 100 subnets metric-type 1<不写默认E2>   (metric 50 不写默认20) 

---进eigrp---
- router eigrp 100
- redistribute ospf 1 metric 10000 100 255 1 1500

---设置环回接口直连路由/静态路由重分布进其他协议    在本身路由器上操作---
- router ... //本身路由器在哪个协议写哪个
- redistribute connected/staic ......

---用route-map做过滤---
- access-list 99 permit 99.99.99.0 0.0.0.255    //先定义
- route-map list1 deny 10
- match ip address 99     //匹配acl99
- router ...
- redistribute..... route-map list1
- route-map list1 permit 20   //permit所有
distribute-list分发列表
同一路由协议之间 一台路由器将路由条目<环回接口>传到另一台路由器
主要用于矢量型路由协议rip eigrp bgp 同一路由协议间的过滤
命令 //需要先用列表定义出来
router eigrp 1 
distribute-list 11<名字> out/in e0/0      //先用acl定义

distribute-list prefix/route-map 11<名字> out/in e0/0     //用前缀列表or路由图定义名字前面要加
passive-interface
主要用于连接主机服务器使用
路由条目参数修改
distance修改管理距离AD
命令
---修改特定网段AD值---

router ospf 1
distance 150 3.3.3.3<ospf中写邻居rid 其他协议写邻居接口> 0.0.0.0 89

策略路由
相当于转发策略

分类
access-group
PBR

设置优先级
负载均衡(分多条路径走)
命令
- access-list 1 permit 10.10.10.1 0.0.0.0
- route-map list1 permit 10
- match ip add 10
- match length 3 200
- set ip (default) next-hop 6.6.6.6 <若不加default流量只往6.6走，加了是优先往6.6走>   //set int e0/0 <设置出接口>
- int ...<in接口>
- ip policy route-map list1 <route-map应用在pbr在路由的in接口>
