## 全局块

- 从配置文件开始到events块之间的内容。
- 主要设置一些nginx服务器整体运行的配置指令。
- 包括运行nginx的用户组、允许生成的worker_processes数、Master PID存放路径、日志存放路径、配置文件的引入
- worker process数越大，可以支持的并发处理量越多

```bash
user www-data;
worker_processes auto;
pid /run/nginx.pid;
error_log /var/log/nginx/error.log;
include /etc/nginx/modules-enabled/*.conf;
```

## events 块

- 这块的指令影响nginx与用户的网络连接。

- 包括是否开启对多worker _processes下的网络连接进行序列化、是否允许同时接收多个网络连接、每个worker_processes可以同时支持的最大连接数。

  ```bash
  #最大连接数
  events {
          worker_connections 768;
          # multi_accept on;
  }
  ```

## http 块

- 包括文件引入、MIME_TYPE定义、日志自定义、连接超时时间、单链接请求数上限。
- centos的server 块包含在http里面

## server 块

- ubuntu放在/etc/nginx/sites-available/* 添加站点就在这个目录下，创建xxx.conf，并软连接到/etc/nginx/sites-enabled/*

```bash
server{
        listen          80;    #外部访问
        server_name     nethub.20050305.xyz;



        location / {
                proxy_pass http://8.148.222.151:8000;  #需要加http/https
                proxy_set_header Host $host;  	 #将get请求中的host 域名信息发给服务器
                proxy_set_header X-Real-IP $remote_addr; #将用户的真实ip发给服务器

        }
}
```

- ### nginx变量：nginx 从请求中自动提取

  - $host — 从用户请求的 Host 头自动获取
  - $remote_addr — 用户真实 IP
  - $scheme — http/https

- ### root & proxy_pass

  - root 绝对路径： 让 Nginx 自己读硬盘文件返回（静态文件服务器），一般放在server全局不放location
  - proxy_pass ip：反向代理
  - 同一个 location 里不能同时生效，Nginx 二选一

- ### 加上ssl证书

  - 多个监听服务不能写在同个server内

  ```bash
  server{
          listen          80;					
          server_name     nethub.20050305.xyz;
  		return 301 https://$host$request_uri;   #重定向到https，301永久重定向状态码
  }
  
  server{
  		listen 			443 ssl;		#https端口
  		server_name     nethub.20050305.xyz;
  		ssl_certificate  /etc/ssl/certs/域名.pem;		#两个证书的路径
  		ssl_certificate_key /etc/letsencrypt/live/域名/fullchain.pem;
  
          location / {
                  proxy_pass http://8.148.222.151:8000;
                  proxy_set_header Host $host;
                  proxy_set_header X-Real-IP $remote_addr;
  
          }
  }
  ```
