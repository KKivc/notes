# 错误日志

- 默认存放在mysql数据目录，默认日志文件名为hostname.err<hostname是主机名>

```sql
#查看日志位置
show variables like 'log_error%';
```

# binlog

- 二进制日志

- `用于数据恢复`、`数据复制`

- 记录了所有DDL语句、DML语句，但是不包括查询语句

- mysql主从复制通过binlog实现

- mysql8默认开启，低版本需要修改配置，win：my.ini       linux：my.cnf

  ## 日志格式

  - STATEMENT
    - 该日志格式在文件中记录的是SQL语句，每一条对数据进行修改的sql都会记录在日志文件中，通过mysqlbinlog工具，可以查看每条语句的文本
    - 主从复制的时候，从库会将日志解析成原文本，并在从库重新执行一次
  - ROW
    - 该日志格式记录的是每一行的数据变更，而不是记录sql语句
  - MIXED
    - 混合STATEMENT和ROW两种格式

```sql
#配置开启binlog日志，日志文件前缀：mysqlbin——>生成文件名：mysqlbin.000001
log_bin=mysqlbin

#配置二进制日志格式
binlog_format=STATEMENT


#查看是否开启
show variables like 'log_bin';

#查看日志格式
show variables like 'binlog_format';

#查看特定的binlog日志
SHOW BINLOG events in 'binlog.000003';

#查看所有binlog文件
show binary logs;

#查看最新日志
show master status;
```

## 数据恢复

```sql
#查看所有binlog文件
show binary logs;

---误删---

#先开启一个新的binlog文件
flush logs;

#使用position方式恢复
show binlog events in 'log_name';

---查找出误删的位置---

#恢复
/usr/bin/mysqlbinlog --start-position=位置 --stop-position=位置 --database=数据库名 /binlog文件的绝对路径 | /usr/bin/mysql -uroot -p密码 -v 数据库



#使用mysqlbinlog时间恢复

#查看时间
mysqlbinlog '/binlog文件的绝对路径'

#恢复
/usr/bin/mysqlbinlog --start-datetime=时间 --stop-datetime=时间 --database=数据库名 /binlog文件的绝对路径 | /usr/bin/mysql -uroot -p密码 -v 数据库

```

# 查询日志

- 记录了客户端的所有操作语句，而binlog不包含查询数据的sql语句
- 默认未开启
- 指令只能当前会话生效，如果要永久一样要改配置   win：my.ini       linux：my.cnf

```sql
#开启查询日志，0：关闭  1：开启
general_log=1;

#设置日志文件名，若没有指定则默认hostname.log
general_log_file=file_name;


#查询是否开启
show variables like 'general_log';
```

# 慢查询日志

- 记录了所有执行时间超过参数long_query_time设置值的所有sql查询语句的日志
- long_query_time默认10s，最小为0

```sql
#开启
slow_query_log=1;

#指定慢查询日志文件名
slow_query_log_file=slow_query.log;

#配置时间限制，超过将认为值慢查询，需要进行日志记录
long_query_time=10;
```
