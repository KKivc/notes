# 读写分离

- 一个master主库，负责写入数据，写库
- 其他slave库从库，负责读取数据，读库
- 主库进行更新时会自动将数据复制到从库，而客户端读取数据，从从库读取
- 可以实现更高的并发量访问，还可以对从服务器进行负载均衡

# 数据备份

- 将主库的数据复制到从库上，相当于一种热备份机制，也就是在主库运行时进行的备份，不会影响服务

# 高可用性

- 数据备份实际是一种冗余机制，这种方式可以换取数据库的高可用性，当服务器出现故障时，可以切换到从服务器上

# 原理

- `slave`会从`master`读取`binlog`进行数据同步

![a319603023e617f3fd95defb3c239dd2_0](%E4%B8%BB%E4%BB%8E%E5%A4%8D%E5%88%B6.assets/a319603023e617f3fd95defb3c239dd2_0.png)

## 复制三步骤

- master将写操作记录到binlog
- slave将master的binlog events拷贝到它的中继日志relay log
- slave重做中继日志中的事件，应用到自己的数据库中。mysql复制时异步的且串行化的，而且重启后从接入点开始复制

# 同步数据一致性问题

## 主从同步要求

- 读库和写库的数据一致（最终一致）
- 写数据必须写到写库
- 读数据必须到读库（不一定）
