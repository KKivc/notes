title：索引

> 索引相当于一个目录，提高查询速度

# 分类

- Hash索引
- B+Tree索引

# 单列索引

- 一个索引只包含单个列，但一个表可以有多个单列索引

## 普通索引

- 没有限制要求，允许再定义索引的列中插入重复值和null的数据

```sql
#创建表时建立
CREATE TABLE if not EXISTS db1.emp2(
  id int ,
  name VARCHAR(10),
  gender VARCHAR(1),
  salary FLOAT,
  index index_name(name)	--给name创建索引，index_name为索引名字
);

#直接创建
create index index_name on emp2(name);

#修改表结构
alter table emp2 add index index_name(name);
```

### 查看索引

```sql
#查看数据库所有索引
select * from mysql.'innodb_index_stats' a where a.'database_name' = '数据库名';

#查看表中所有索引
select * from mysql.'innodb_index_stats' a where a.'database_name' = '数据库名' and a.table_name like '%表名%';

#查看表中所有索引
show index from 表名;
```

### 删除索引

```sql
drop index 索引名 on 表名;

#or
alter table 表名 drop index 索引名;
```

## 唯一索引

- 索引列的值必须唯一，但是允许null

```sql
#创建表时指定
CREATE TABLE if not EXISTS db1.emp2(
  id int ,
  name VARCHAR(10),
  gender VARCHAR(1),
  salary FLOAT,
  unique index_name(name)	
);

#直接创建
create unique index index_name on emp2(name);

#修改表结构
alter table emp2 add unique index_name(name);
```

## 主键索引

- 当创建主键时，就会自动在主键列上建立一个索引
- 必须唯一且不为null

# 组合索引

- 建立索引时可以使用多个字段

```sql
#给手机号和名字添加普通索引		两列索引的值不能完全一样
create index index_phone_name emp(phone,name);
```

# 全文索引

- 相似度查询
- 数据量大不适用
