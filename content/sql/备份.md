# 备份

- mysqldump命令执行时，将数据库备份成一个`文本文件`啊，包含`create`和`insert`语句，使用这些语句可以重新创建表和插入数据

```sql
#语法
mysqldump -u 用户名称 -h 主机名称 -p密码 数据库名字 > 备份文件名字.sql
```

# 恢复

- 通过insert语句插入备份的数据
- 如果备份文件里面包含创建数据库的语句，则恢复的时候不需要指定数据库名称

```sql
mysql -u root 数据库名 < backup.sql
```

# 误删操作

- 使用delete语句误删数据行
- drop table / truncate table 语句误删数据库
- drop database语句误删数据库
- rm命令误删整个mysql实例

## 解决

### delete误删行

#### 数据恢复

- 使用`flashback工具`恢复
- 原理：修改binlog内容，拿回原库重放。如果误删数据涉及多个事务的话，需要将事务顺序调过来在执行
- 使用前提：`binlog_format=row`和`binlog_row_image=FULL`

#### 预防

- 代码上线前，必须sql审查、审计
- 打开安全模式，把sql_safe_update参数设置为on

### truncate/drop误删库/表

#### 数据恢复

- 需要使用`全量备份`和`增量日志`结合的方式，有定期的全量备份，并且实时备份binlog

- 取最近一次全量备份
- 用备份恢复出一个`临时库`,这里选择临时库，而不是直接操作主库
- 取出binlog日志
- 剔除误删除的语句外，其他语句全部应用带临时库
- 最后恢复到主库

#### 预防

- 权限分离
- 制定操作规范
- 设置延迟复制备库

### 误删mysql实例

- 不用担心rm删除数据，因为只是删除其中一个节点数据，HA系统会选出一个新的主库
